FPGA für M86 Package:

Mit dem Objekt "FPGA" wird eine FPGA Datei zum aktuellen Package hinzugefügt. 
Zusätzlich zum File müssen Information zum FPGA selber (dualboot, type,..) ausgefüllt
werden. Diese werden zum einem für mögliche Hilfestellungen wie bei der Startadresse
verwendet (Default Adresse wird selber eingefügt). Aber auch direkt in das Package 
integriert.

Propterties:
--descripion		...Beschreibung der FPGA Datei (wird nicht verwendet)
--outputFile_name	...Name folgender generierter Dateien: "h86", "mch", "ver"
--inputFile_directory	...Dateipfad zum Binärfile der Logik (wichtig: Binärdatei!)
--fpgatype		...Verwendeter FPGA Hersteller bzw. Produktfamilie
--designnumber	...Designnummer der Logik laut Logikreferenzliste
--revision		...Hauptversion der Logik
--testversion		...Testversion der Logik
--dualboot		...Auswahl der Dualboot-Funktion
--flash_size		...Größe des Config-Flashs
--start_addr		...Startadresse des Working Codes im Flash 
		   (Bei Lattice vorgegeben!)

Beispiel:

description		= new FPGA
outputFile_name	= 14800501
fpgatype		= Lattice ECP2
inputFile_directory	= C:\Documents\PackGen\SAI205_v19_working.bit
designnumber	= 1480
revision		= 05
testversion		= 01
dualboot		= enable
flash_size		= 16Mbit
start_addr		= 0x010000

Der Name der "mch"-Datei wird beim Öffnen der Package-Datei im SolutionCenter 
angezeigt. Deshalb sollte hier der Name der entsprechenden Komponente angegeben 
werden (bei Logik: derselbe Name, wie die "bin"-Datei zusammengestellt aus 
Designnummer, Hauptversion und Testversion)

->Bei CPU Modul darf der Name NICHT derselbe wie der des Eingangsfiles sein 
(wenn sie im selben Ordner angelegt werden). Ansonsten wird das Ursprüngliche 
File Überschrieben!
